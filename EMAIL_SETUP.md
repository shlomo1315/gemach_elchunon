# 📧 הפעלת התראות מייל — גמ"ח זכרון אהרן

המערכת שולחת מייל HTML מעוצב **בכל פעולה** (הפקדה, משיכה, חבר חדש, עריכה,
מחיקה, שיקים, אישור בקשות, ייבוא):

- **למנהל** — תמיד, על כל פעולה.
- **לחבר** — רק אם הפעולה קשורה אליו **ויש לו כתובת מייל שמורה** בכרטיס.

> בלי ההגדרה למטה המערכת עובדת רגיל — פשוט לא נשלחים מיילים. שום דבר לא נשבר.

---

## הפעלה — 4 צעדים (חד-פעמי)

### 1. חשבון Resend (חינם)
- נכנסים ל-https://resend.com ונרשמים (תוכנית חינם ~3,000 מייל/חודש).
- בתפריט **API Keys** → **Create API Key** → מעתיקים את המפתח (מתחיל ב-`re_`).

### 2. כתובת שולח
- **לבדיקה מהירה:** אפשר להשתמש בכתובת `onboarding@resend.dev` (עובד מיד, בלי דומיין).
- **לכתובת אמיתית** (מומלץ בהמשך): ב-Resend → **Domains** → מוסיפים ומאמתים דומיין,
  ואז אפשר לשלוח מ-`noreply@your-domain.com`.

### 3. הוספת משתני סביבה ב-Vercel
ב-**Vercel → הפרויקט → Settings → Environment Variables**, מוסיפים שלושה:

| שם | ערך |
|----|-----|
| `RESEND_API_KEY` | המפתח שהעתקת (`re_...`) |
| `ADMIN_EMAIL` | המייל שלך — לשם יגיעו ההתראות |
| `EMAIL_FROM` | למשל `גמ"ח זכרון אהרן <onboarding@resend.dev>` |

### 4. Redeploy
ב-Vercel → **Deployments** → על הפריסה האחרונה → **Redeploy**
(כדי שמשתני הסביבה ייטענו).

---

## איך זה עובד טכנית
- כל פעולה במערכת קוראת ל-`notify()` (צד לקוח) שמפעיל את `/api/notify` (צד שרת).
- נקודת הקצה מאמתת שהמשתמש מחובר, שולפת את מייל החבר לפי המזהה, ושולחת דרך Resend.
- השליחה היא "שגר ושכח" — אם המייל נכשל, **הפעולה במערכת כבר נשמרה** והממשק לא נתקע.

## כדי שגם החברים יקבלו מייל
צריך שלכרטיס החבר תהיה כתובת מייל. אפשר להוסיף מייל בכרטיס החבר.
חברים בלי מייל — רק המנהל יקבל התראה לגביהם.
